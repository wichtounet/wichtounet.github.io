Do not use relative path with LogBack

Posted: 2010-08-12 07:53   |   More posts about Java Java Tips

A little tip that can be useful and save a lot of time : Do not use relative path with LogBack.

I wondered why this little LogBack configuration didn't work :

[xml]<?xml version="1.0" encoding="UTF-8" ?>

<configuration> <contextName>JTheque</contextName>

&lt;appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender"&gt;
    &lt;file&gt;logs/jtheque.log&lt;/file&gt;

    &lt;rollingPolicy class="ch.qos.logback.core.rolling.FixedWindowRollingPolicy"&gt;
        &lt;FileNamePattern&gt;logs/jtheque.%i.log.zip&lt;/FileNamePattern&gt;
        &lt;MinIndex&gt;1&lt;/MinIndex&gt;
        &lt;MaxIndex&gt;5&lt;/MaxIndex&gt;
    &lt;/rollingPolicy&gt;

    &lt;triggeringPolicy class="ch.qos.logback.core.rolling.SizeBasedTriggeringPolicy"&gt;
        &lt;MaxFileSize&gt;5MB&lt;/MaxFileSize&gt;
    &lt;/triggeringPolicy&gt;

    &lt;layout class="ch.qos.logback.classic.PatternLayout"&gt;
        &lt;Pattern&gt;%d{HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n&lt;/Pattern&gt;
    &lt;/layout&gt;
&lt;/appender&gt;

&lt;root level="DEBUG"&gt;
    &lt;appender-ref ref="FILE"/&gt;
&lt;/root&gt;

</configuration>[/xml]

No file were written. I searched during a long time and after that tested with an absolute path and it worked really well. But absolute path is not very good. But, you can use system properties in the configuration, so I used user.dir to make the think works :

[xml] ... <file>${user.dir}/logs/jtheque.log</file>

    &lt;rollingPolicy class="ch.qos.logback.core.rolling.FixedWindowRollingPolicy"&gt;
        &lt;FileNamePattern&gt;${user.dir}/logs/jtheque.%i.log.zip&lt;/FileNamePattern&gt;
        &lt;MinIndex&gt;1&lt;/MinIndex&gt;
        &lt;MaxIndex&gt;5&lt;/MaxIndex&gt;
    &lt;/rollingPolicy&gt;
    ...

[/xml]

And this time, it works well !

Hope this will be useful to somebody.
